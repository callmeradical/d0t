Dotfiles, declared.

d0t is a dotfiles manager with an explicit manifest DSL. Every file, hook, and package is declared in a d0tfile. Nothing is inferred from directory layout or filename conventions.

Get started GitHub → 
# base/d0tfile

link   home/.zshrc
link   xdg/nvim
copy   home/.ssh/config  mode=0600
tmpl   home/.gitconfig

brew   ripgrep neovim bat fzf lazygit
cask   ghostty raycast

hook   post-apply hooks/post-apply.sh
📋

Explicit by design

The d0tfile is the source of truth. Files in your repo are just storage — nothing is auto-discovered.

🔀

Profile layering

Stack base → os/darwin → hosts/work-mbp. Later profiles override earlier ones per target path.

🔑

Secret injection

Reference 1Password, macOS Keychain, or env vars directly in templates. Secrets never touch the repo.

📦

Package management

Declare brew formulae, casks, taps, apt packages, and mas apps alongside your config files.

⚡

Idempotent

Re-running d0t apply is always safe. State is tracked so d0t remove cleanly undoes everything.

🔧

Single static binary

No runtime dependencies. One Go binary. Works on macOS and Linux. Bootstraps fresh machines from zero.

Install

Build from source

git clone https://github.com/callmeradical/d0t ~/.d0t-src
go build -o ~/.local/bin/d0t ~/.d0t-src/cmd/d0t

go install

go install github.com/callmeradical/d0t/cmd/d0t@latest

Requires Go 1.21+. Make sure ~/.local/bin (or your $GOPATH/bin) is in your $PATH.

Verify

d0t
# usage: d0t [global flags] <command> [args]
Getting started

Quick start

# 1. Scaffold a new repo
d0t init                     # creates ~/.d0t with starter d0tfile

# 2. Adopt existing config files into the repo
d0t adopt ~/.zshrc           # moves file in, replaces with symlink
d0t adopt ~/.gitconfig

# 3. Preview what apply will do
d0t plan

# 4. Apply — link files, run hooks, install packages
d0t apply

The repo defaults to ~/.d0t. Override with --repo /path or the $D0T_REPO environment variable.

Tip: Set export D0T_REPO="$HOME/.d0t" in your .zshenv so you never need to pass --repo.

Getting started

Core concepts

The d0tfile

Every profile directory contains a d0tfile. This is d0t's source of truth — like a Makefile or a Chef recipe. Only what is declared gets managed. Source files sitting in home/ or xdg/ are ignored unless they appear in the d0tfile.

Profiles

A profile is a directory inside your repo. Profiles layer in order: base → os/darwin → hosts/work-mbp. For any target path, the last profile to declare it wins. Hooks and packages accumulate across all profiles.

Primitives

There are six primitives: link, copy, template, fragment, exec (hooks), and pkg. Each is declared with a keyword on one line of the d0tfile.

State

d0t tracks every managed target in ~/.local/state/d0t/state.json. This lets d0t remove safely undo exactly what d0t apply created, and lets d0t status detect drift.

The d0tfile

DSL syntax

The d0tfile is a line-oriented manifest. One declaration per line. # starts a comment (full-line or inline). Blank lines are ignored. Keywords are case-insensitive.

# base/d0tfile — full example

# ── file primitives ──────────────────────────────────
link     home/.zshrc
link     home/.zshenv
link     xdg/nvim                   # links the whole directory
copy     home/.ssh/config  mode=0600
tmpl     home/.gitconfig             # rendered with Go text/template
fragment fragments/path.fragment      # injects block into target file

# ── hooks ────────────────────────────────────────────
hook     pre-apply  hooks/pre-apply.sh
hook     post-apply hooks/post-apply.sh  optional

# ── packages ─────────────────────────────────────────
tap      homebrew/cask-fonts
brew     ripgrep neovim bat fzf lazygit starship tmux
cask     ghostty raycast font-monaspice-nerd-font
apt      ripgrep neovim bat
mas      497799835  Xcode

Source paths

Source paths are relative to the profile directory. The leading component (home/, xdg/, etc.) selects the filesystem root and is mapped to the target location:

Source prefixMaps to
home/$HOME
xdg/$XDG_CONFIG_HOME (default ~/.config)
etc//etc
fragments/target declared in file frontmatter

Override the target explicitly with target=:

link  home/.zshrc  target=~/.config/zsh/.zshrc
The d0tfile

File primitives

link <source> [target=<path>]

Create a symlink at the target pointing to the source in the repo. Live edits in the repo are immediately visible. Default for most files.

copy <source> [mode=<octal>] [target=<path>]

Copy bytes to the target. Use for files that must not be symlinks — apps that rewrite their config, or files needing chmod 600 like ~/.ssh/config.

tmpl <source> [target=<path>]

Render the source with Go text/template and write to the target. Variables come from vars.toml. Also available as template.

fragment <source> [target=<path>] [marker=<id>]

Inject a managed block into a file d0t doesn't own. The block is delimited by marker comments and can be updated or removed without touching the rest of the file.

link

The default primitive. Edits in the repo are immediately live since the target is a symlink pointing into the repo. If a file already exists at the target and is not a d0t-managed symlink, apply fails unless you pass --adopt (backs up the existing file) or --force (overwrites without backup).

copy

Copies the source bytes atomically to the target. The mode= option sets the target's permission bits in octal — the source file must also have those bits set (d0t preserves source mode). Change detection is by SHA-256 hash so unchanged files are never written.

copy  home/.ssh/config   mode=0600
copy  home/.gnupg/gpg.conf  mode=0600

tmpl / template

Renders the source through Go's text/template. Variables are merged from all active profiles' vars.toml files (later profiles override earlier ones) and are accessed as .Vars.<key>. Built-in variables (.Host, .OS, .User, .Home) are always available.

# base/home/.gitconfig.tmpl
[user]
  name  = {{ .User }}
  email = {{ .Vars.email }}
[core]
  editor = {{ default .Vars.editor "vim" }}

fragment

The source file starts with a header line declaring the target and a unique marker:

# d0t: target=~/.zshrc marker=local-bin
export PATH="$HOME/.local/bin:$PATH"

d0t injects — or replaces — the managed block in the target file:

# >>> d0t:local-bin >>>
export PATH="$HOME/.local/bin:$PATH"
# <<< d0t:local-bin <<<

Comment style is auto-detected from the target file extension (# default, // for Go/JS/TS, -- for Lua/SQL).

The d0tfile

The d0tfile

Hooks

Hook scripts run at lifecycle phases. The script path is relative to the profile directory.

hook  pre-apply   hooks/pre-apply.sh
hook  post-apply  hooks/post-apply.sh
hook  post-apply  hooks/reload-shell.sh  optional
hook  pre-remove  hooks/pre-remove.sh
hook  post-remove hooks/post-remove.sh

Multiple hooks for the same phase are allowed and run in declaration order. Adding optional means a non-zero exit is logged but does not abort the apply.

Profiles

Profile layering

Profiles are directories inside the repo. The default resolution order is:

base/           # always applied
os/darwin/      # applied on macOS (runtime.GOOS == "darwin")
os/linux/       # applied on Linux
hosts/work-mbp/ # applied when hostname == "work-mbp"

For any target path, the last profile that declares it wins. Hooks and packages accumulate from all active profiles.

Profiles without a d0tfile are silently skipped — they contribute nothing to the plan.

Override the profile stack

In d0t.toml:

profiles = ["base", "os/darwin", "work", "hosts/work-mbp"]

Or at the command line:

d0t --profile base,work apply

Example: work vs home

repo/
├── base/d0tfile         # shared config
├── os/darwin/d0tfile    # macOS-specific additions
└── hosts/
    ├── home-mbp/d0tfile # personal machine extras
    └── work-mbp/d0tfile # work machine extras + overrides

Features

Secret management

The secret() template function resolves secrets at render time from a configured backend. Secrets never touch the repo or the state file.

# base/home/.gitconfig.tmpl
[user]
  email      = {{ secret "op://Personal/GitHub/email" }}
  signingkey = {{ secret "keychain://gpg/signing-key" }}

[github]
  token = {{ secret "env://GITHUB_TOKEN" }}

Key prefixes

PrefixBackendExample
op://1Password CLIop://Personal/Item/field
keychain://macOS Keychainkeychain://service/account
env://Environment variableenv://MY_TOKEN
pass://passpass://path/to/secret
(bare)Configured defaultgithub-token

Configure the default backend

# d0t.toml
[secrets]
backend = "op"       # 1Password as default for bare keys
# backend = "keychain"
# backend = "env"
# backend = "pass"

Caching. Each unique key is resolved once per apply run. op read fires at most once per key regardless of how many templates reference it.

Features

Templates

Template files (tmpl keyword) use Go's text/template syntax. Variables come from vars.toml files in each active profile, merged in profile order.

# base/vars.toml
email  = "you@example.com"
editor = "nvim"
# hosts/work-mbp/vars.toml — overrides base
email = "you@company.com"

Features

Fragments

Fragments let d0t manage a block inside a file it doesn't own. Useful for appending to system files or files managed by other tools.

Fragment file format

The source file starts with a single header line:

# d0t: target=~/.zshrc marker=local-bin
export PATH="$HOME/.local/bin:$PATH"

target is the file to inject into. marker is a unique ID for this block within that file — it's how d0t finds and replaces the block on subsequent applies.

The managed block

d0t writes the block surrounded by marker comments:

# >>> d0t:local-bin >>>
export PATH="$HOME/.local/bin:$PATH"
# <<< d0t:local-bin <<<

Subsequent applies replace the block in place. d0t remove deletes the block, leaving everything else in the file untouched.

d0t.toml

An optional file at the repo root. All fields are optional.

# d0t.toml

# Override the default profile resolution order
profiles = ["base", "os/darwin", "work", "hosts/work-mbp"]

# Add custom filesystem roots
[roots]
caches = "${XDG_CACHE_HOME:-$HOME/.cache}"
data   = "${XDG_DATA_HOME:-$HOME/.local/share}"

# Secret backend for {{ secret "bare-key" }} calls
[secrets]
backend = "op"

# Source path for d0t update (self-rebuild)
[self]
source = "~/Dev/d0t"
