Before you start

What's changing

You currently manage dotfiles with GNU stow: each tool has its own directory that mirrors $HOME, and stow creates the symlinks. d0t replaces stow with an explicit manifest (d0tfile) — you declare every file, hook, and package in one place. Nothing is inferred from directory layout.

Conceptstowd0t
DiscoveryWalk package dir, mirror to $HOMEOnly what's in d0tfile
Config dirsnvim/.config/nvim/base/xdg/nvim/
Home fileszsh/.zshrcbase/home/.zshrc
PackagesSeparate Brewfile (not stowed)brew/cask/tap in d0tfile
Hookssetup.sh / fix_dotfiles.shhook post-apply in d0tfile
SecretsNot handled{{ secret "op://..." }} in templates
Host-specificNot handled (single machine)hosts/<hostname>/d0tfile

Your existing repo stays at ~/Documents/3_Resources/dotfiles and keeps its GitHub remote. You're adding a base/ profile directory and reorganising files into it. The old stow packages are deleted after the cutover.

Step 1

Install d0t

Build from the source repo you already have:

go build -o ~/.local/bin/d0t ~/Dev/d0t/cmd/d0t

Confirm it works:

d0t
# usage: d0t [global flags] <command> [args]

Make sure ~/.local/bin is in your $PATH. It should be — it's already in your .zshrc via the Antigravity PATH export.

Write the post-apply hook

This replaces the chmod 600 ~/.ssh/config line from the old setup.sh. The copy primitive already preserves the source mode, so this hook is a good place to put any other post-install steps.

cat > base/hooks/post-apply.sh <<'EOF'
#!/bin/sh
set -e

echo "d0t: post-apply on $D0T_HOST"

# TPM: install tmux plugins if not already installed
if [ ! -d ~/.tmux/plugins/tpm ]; then
  git clone https://github.com/tmux-plugins/tpm ~/.tmux/plugins/tpm
fi
EOF
chmod +x base/hooks/post-apply.sh

Add d0t.toml (optional but recommended)

cat > d0t.toml <<'EOF'
# d0t repo config — see README for full reference

# Uncomment and set to "op" once you're ready to use 1Password for secrets
# [secrets]
# backend = "op"
EOF
Step 4

Verify with d0t plan

d0t plan is a dry-run — it reads the d0tfile, computes what would change, and prints it. Nothing is written to disk. Run it as many times as you like until the output looks right.

d0t --repo ~/Documents/3_Resources/dotfiles plan

You should see something like this (the exact paths will show your hostname):

repo:     /Users/lars/Documents/3_Resources/dotfiles
host:     Aurelius (darwin/arm64)
profiles: base

  create   link /Users/lars/.zshrc <- base/home/.zshrc
  create   link /Users/lars/.zshenv <- base/home/.zshenv
  ...
  create   copy /Users/lars/.ssh/config <- base/home/.ssh/config
  create   exec [post-apply] post-apply.sh
  ...
Expect conflicts

Because stow symlinks already exist at most targets, plan will show conflict not create for those paths. That's correct — they point to the old stow locations. The conflicts are resolved in the next step with --adopt.

Common issues to look for

  • conflict — target exists and is not a d0t-managed symlink (expected for current stow symlinks)
  • ERROR — source file doesn't exist at the declared path (check your cp commands in Step 2)

If you see an ERROR, find the missing file and re-copy it, then re-run plan.

Step 5 — point of no return

Cut over

Read before running

This step removes the stow symlinks and replaces them with d0t-managed ones. Have a terminal window open. Your shell, editor, and terminal emulator will continue working through the transition — their configs are symlinks to files that don't move.

1 — Remove stow symlinks

We unstow everything first so the paths are clear for d0t to manage.

cd ~/Documents/3_Resources/dotfiles

# Remove stow symlinks (does NOT delete the files in the repo)
stow -D ghostty nushell nvim ssh starship tmux wezterm zsh

# Verify they're gone
ls -la ~/.config/nvim ~/.zshrc ~/.tmux.conf 2>&1

You should see "No such file or directory" or dangling symlinks. That's expected.

television was never stowed, so no need to unstow it.

2 — Apply with d0t

d0t --repo ~/Documents/3_Resources/dotfiles apply --no-pkg --adopt

The --adopt flag backs up any file that's in the way before replacing it. --no-pkg skips brew installs for now — packages are already installed on your machine.

3 — Verify everything is wired up

# All should show d0t-managed symlinks
ls -la ~/.zshrc ~/.zshenv ~/.tmux.conf
ls -la ~/.config/{ghostty,nvim,nushell,starship.toml,tmux,wezterm}

# ssh/config should be a regular file (copy, not symlink)
ls -la ~/.ssh/config
stat -f "%Mp%Lp %N" ~/.ssh/config   # should show 0600

# Television should now be wired
ls -la ~/.config/television/config.toml

# Check d0t status — should show everything as ok
d0t --repo ~/Documents/3_Resources/dotfiles status

4 — Open a new shell and confirm

exec zsh   # or open a new tab in ghostty/wezterm

If your prompt loads, aliases work, and neovim opens — you're done with the cutover.

Step 6 — optional

Wire up 1Password secrets

You have 1password-cli@beta installed. Once you're signed in to op, you can reference secrets directly in any .tmpl file instead of hardcoding them or leaving them out.

Enable the backend

# d0t.toml — uncomment the secrets section
cat > d0t.toml <<'EOF'
[secrets]
backend = "op"
EOF

Example: .gitconfig with 1Password

Rename the file and add template calls:

mv base/home/.gitconfig base/home/.gitconfig.tmpl
# base/home/.gitconfig.tmpl
[user]
    name  = Lars Cromley
    email = {{ secret "op://Personal/GitHub/email" }}

[github]
    user  = callmeradical

[credential]
    helper = osxkeychain

[core]
    editor = nvim
    excludesfile = ~/.gitignore_global

Then update base/d0tfile to use tmpl instead of link:

# in base/d0tfile, replace:
link   home/.gitconfig
# with:
tmpl   home/.gitconfig

Sign in and test

op signin
d0t --repo ~/Documents/3_Resources/dotfiles plan

Secrets are cached per apply run — op read fires once per unique key even if many templates reference the same path.

Make d0t the default

Set $D0T_REPO in your .zshenv so you never need --repo:

After that, all d0t commands work without --repo:

d0t status
d0t plan
d0t apply
Step 8 — bonus

Bootstrapping a new machine

The whole point of managing dotfiles is making a new machine feel like home in minutes. With d0t this is now one script.

The bootstrap sequence

# 1. Install Homebrew
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 2. Install Go (needed to build d0t)
brew install go

# 3. Clone your dotfiles
git clone https://github.com/callmeradical/dotfiles ~/Documents/3_Resources/dotfiles

# 4. Build d0t
mkdir -p ~/.local/bin
go build -o ~/.local/bin/d0t ~/Documents/3_Resources/dotfiles/../Dev/d0t/cmd/d0t

# 5. Apply — this installs all packages AND links all configs
export D0T_REPO=~/Documents/3_Resources/dotfiles
~/.local/bin/d0t apply --adopt
What apply does on a fresh machine

Runs taps → installs formulae → installs casks → creates all symlinks and copies → runs the post-apply hook (clones tpm). One command, fully converged.

Tip: keep d0t in the d0tfile

Once you publish d0t or install it via your tap, you can add it to base/d0tfile so it installs itself on a new machine:

brew   anomalyco/tap/opencode  # already there
# add d0t here once it's published to a tap

Per-machine overrides

For a work machine with different config, add a host profile:

hosts/
└── work-mbp/
    ├── d0tfile          # declares overrides and additions
    └── home/
        └── .zshenv      # sets LOCATION=work, different PATH, etc.
# hosts/work-mbp/d0tfile
link   home/.zshenv     # overrides base/.zshenv on this machine

# work-only packages
brew   awscli terraform vault
cask   zoom slack
